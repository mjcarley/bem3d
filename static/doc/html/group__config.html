Detailed Description

BEM3D codes can be configured at run-time using glib key files and the following functions.

Enumeration Type Documentation

enum BEM3DSolver

Solver selection to choose between, for example, direct and Fast Multipole Method in configuration files.

gint bem3d_configuration_add_identifier ( const gchar *  id,
gpointer  v 
)

Add an identifier to the configuration data. This allows the identifier to be recognized by the bem3d_configuration_ functions. For example,

bem3d_configuration_add_identifier("bem3d_quadrature_polar", bem3d_quadrature_rule_polar) ;

adds the identifier "bem3d_quadrature_polar" which refers to the quadrature function of the same name, allowing a key file to contain a configuration line of the form

quadrature[1] = bem3d_quadrature_polar(0.0,16,16)

Parameters
idname of identifier;
va pointer to the identified object
Returns
::BEM3D_SUCCESS on success

gint bem3d_configuration_set ( BEM3DConfiguration *  c,
gpointer  k 
)

Set fields of a BEM3DConfiguration from a key file, overwriting previous settings. Settings are read from the "BEM3D" group of the file, to allow codes to keep all their settings in one place. Values not in the "BEM3D" group are ignored.

Parameters
ca BEM3DConfiguration;
ka GKeyFile containing settings for c.

::BEM3D_SUCCESS on success.

Write generic configuration data containing all possible settings, which can be used as a template for a configuration file. Output is written in the form:

[Group name] prefix [setting] = [variable name] prefix [setting] = [variable name] prefix [setting] = [variable name]
