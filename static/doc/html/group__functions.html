Detailed Description

Application of analytical functions to data on meshes. The underlying function handler is GNU libmatheval and functions should conform to that syntax, including the built-in functions. Reserved variable names are x, y, z for node coordinates, u, v, w for components of node velocity, nx, ny, nz for node normal, i for node index, and mesh for mesh index (when multiple meshes are passed to the function evaluation). In integral evaluations, refers to the node coordinates in an integrand, X, Y, and Z are reserved for node coordinates at an evaluation point, NX, NY, and NZ for the corresponding normal, and I for the node index.

Functions are applied to data at each node of a mesh with input and output data in BEM3DMeshData structs. Input and output data are specified in functions as f and g, with components given as f[0], etc. Output data are given in f.

Apply a function to surface data specified as BEM3DMesh. The function may use any of the reserved words which apply to a BEM3DFunction and entries for the supplied data blocks f and g (if not NULL) with reference to `f[0]', `f[1]', etc. for elements of data and `dfdx[0]' etc. for gradient terms, which are computed as required at each node. If a second data block is given, it may be included in the function as `g[0]' etc. On output, f will contain the results of the applied function.

Parameters
funcBEM3DFunction to apply;
mBEM3DMesh for surface;
fa BEM3DMeshData block containing data for mesh (must not be NULL);
ga BEM3DMeshData block containing supplementary data (may be NULL);
Returns
0 on success.
