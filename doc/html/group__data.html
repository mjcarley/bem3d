BEM3DMeshData *  f2,
gboolean  strict

Merge two data blocks into a single block. This is intended for merging data generated for two disjoint meshes, with different indices. If an index appears in both blocks the behaviour is undefined. The output block will be sized to accomodate all data from both inputs, with zero padding where required.

Parameters
f1an input BEM3DMeshData;
f2another input BEM3DMeshData;
strict(not currently used).
Returns
a new BEM3DMeshData containing the data from f1 and f2.

gint bem3d_mesh_data_write ( BEM3DMeshData *  f,
FILE *  fp,
gchar *  header 
)

Write a BEM3DMeshData block to a file. The output format is one line containing:

NP NE BEM3DMeshData

where NP is the number of nodes in the block and NE is the number of numerical entries per point, followed by NP lines each containing NE numerical entries. If header is not NULL, it is used in place of the standard first line. If it contains C formatting strings d, these generate NP and NE.

Parameters
fBEM3DMeshData block;
fpfile pointer;
headerfirst line of output file (may be NULL)
Returns
0 on success.

gint bem3d_mesh_data_write_weights ( BEM3DMeshData *  f,
FILE *  fp,
gint *  fields,
gint  nfields 
)

Write a block of data in the form of a single row suitable for use in a weighting matrix, e.g. of a type generated by bem3d_function_integral_weights. Data are written on one row in the form

index f0 f1 f2 index f0 f1 f2 ...

with any entries with all data equal to zero neglected, effectively treating the data as a sparse matrix.

Parameters
fdata to write;
fpfile pointer for output;
fieldsindices of fields to write to output (must all be less than the number of fields per node in f);
nfieldsnumber of fields to write.
Returns
0 on success
