\subsection{Detailed Description}
B\+E\+M3\+D contains a simple interpreter, using the libmatheval library ({\tt http\+://www.\+gnu.\+org/software/libmatheval/}) of Aleksandar Samardzic, which can be used to analytically specify the motion of the nodes of a \doxyref{B\+E\+M3\+D\+Mesh}{p.}{structBEM3DMesh} and analytically derive the surface velocity for use in the boundary condition. Motion definitions are defined using equations of the form `x = ...\textquotesingle{}, where the right hand side may include other user-\/defined functions and constants. See \doxyref{bem3d\+\_\+motion\+\_\+variable\+\_\+add}{p.}{group__motion_gabac941c41da8d12510a5faf154176d36} to see how this works.

Add a variable definition to a \doxyref{B\+E\+M3\+D\+Motion}{p.}{structBEM3DMotion}, in the form {\itshape var} = {\itshape def}. The variable name {\itshape var} may not be one of the reserved tokens but may otherwise be any valid libmatheval token. The variables which must be defined (and are given default definitions in \doxyref{B\+E\+M3\+D\+Motion}{p.}{structBEM3DMotion}) are $x$, $y$, $z$ which will be evaluated to give the node position and $u$, $v$ and $w$ which will give the surface velocity, other than that due to surface motion (e.\+g. in scattering problems). Reserved tokens are $x0$, $y0$, $z0$, the position of a node on the reference \doxyref{B\+E\+M3\+D\+Mesh}{p.}{structBEM3DMesh}, $nx$, $ny$, $nz$, $nx0$, $ny0$, $nz0$, the components of the local surface normal on the mesh and the reference mesh respectively, $t$, time, $i$, the node index and $j$, the complex variable.
